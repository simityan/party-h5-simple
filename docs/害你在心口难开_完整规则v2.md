# 害你在心口难开·动作版 — 完整规则文档（v2定稿）

> 合并自v1基础规则 + v2补充修改，冲突处以v2为准  
> 更新时间：2026-06-07

---

## 一句话定义

**害你在心口难开 · 动作版** —— 每人手持秘密任务，偷偷引导他人做出指定行为，完成后对方接受惩罚、自己得分；你可以随时质疑识破他人的暗算来反杀得分；最终积分垫底的玩家接受团队惩罚。

## 一、游戏流程总览

```
后台创建游戏 → 设定人数、时间、团队奖励/惩罚 → 生成二维码/链接
        ↓
玩家扫码进入 → 输入昵称 → 等待人齐
        ↓
人齐后游戏自动开始 / 到达预设开始时间自动开始
        ↓
每位玩家获得3条秘密任务（🟢简单 + 🟡中等 + 🔴困难），每条附带对应难度惩罚
目标独立随机分配（简单1名目标/中等1目标+1次要/困难1目标+3次要）
初始3次刷新次数
        ↓
  ┌─────────────────── 游戏进行中 ───────────────────┐
  │                                                   │
  │  【执行任务】                                     │
  │  线下偷偷引导目标做出指定行为                      │
  │  完成后点击【完成】→ 二次确认 → 发送至目标玩家     │
  │  目标玩家确认✅ → 得分 + 翻牌展示惩罚              │
  │  目标玩家否认❌ → 触发质疑流程，执行者受罚         │
  │                                                   │
  │  【质疑反杀】                                     │
  │  点击【质疑】按钮 → 选择目标玩家 → 输入任务内容   │
  │  系统自动文本匹配（≥75%相似度=质疑成功）          │
  │  质疑成功：任务执行者受罚，双方各+1刷新次数       │
  │                                                   │
  │  【刷新任务】                                     │
  │  消耗1次刷新次数 → 三卡同时刷新 → 目标重新随机    │
  │  刷新次数为0时不可刷新                            │
  │                                                   │
  │  【动态流】                                       │
  │  实时滚动：完成的任务 + 质疑成功的记录             │
  │  匿名爆料：至少2条，不足自动补足                   │
  │                                                   │
  └───────────────────────────────────────────────────┘
        ↓
游戏结束（全员点结束 / 预设时间到）→ 直接结算
        ↓
公布排名 → 公开未完成任务 → 精彩回放 → 勋章颁发 → 能力图展示 → 惩罚/奖励执行
```

## 二、角色与入口

### 2.1 角色

| 角色 | 说明 |
|------|------|
| **后台** | 创建游戏、设定参数、生成入口 |
| **玩家** | 扫码进入、输入昵称、参与游戏（4-8人） |

### 2.2 玩家进入流程

1. 扫描二维码或点击链接进入H5
2. 首次进入：输入昵称，系统通过微信OpenID绑定唯一身份
3. 再次进入：自动识别OpenID，读取当前游戏进度
4. 等待参与人数达到设定人数 → 游戏开始

### 2.3 中途规则

- 参与人数未满前，随时可以加入
- 不存在退出机制，关闭/离开页面不影响游戏
- 重新打开自动恢复进度

## 三、任务体系

### 3.1 任务结构

每条任务包含：

| 字段 | 说明 |
|------|------|
| 任务内容 | 需要引导目标做出的行为描述 |
| 难度 | 🟢简单 / 🟡中等 / 🔴困难 / 💀超难 |
| 积分 | 🟢+1 / 🟡+2 / 🔴+3 / 💀+5 |
| 绑定惩罚 | 同难度随机绑定的惩罚内容 |

### 3.2 目标指定规则（v2重构）

| 难度 | 目标数量 | 任务文本模板 | 示例 |
|------|----------|------------|------|
| 🟢 简单 | 1名目标玩家 | 让目标玩家…… | 让目标玩家对你说"你真好" |
| 🟡 中等 | 1名目标玩家 + 1名次要玩家 | 让目标玩家及次要玩家…… | 让目标玩家及次要玩家互相碰杯 |
| 🔴 困难 | 1名目标玩家 + 3名次要玩家 | 让目标玩家及次要玩家们…… | 让目标玩家及次要玩家们依次给你递一次东西 |
| 💀 超难 | 所有其他玩家 | 让所有玩家…… | 让所有玩家主动给你敬酒 |

**目标分配规则：**
- 每个玩家的3张卡各有独立目标，**不可选自己为目标**
- 主要目标在卡牌外统一展示
- 次要目标在对应卡牌内展示
- 3张卡的主要目标可以相同，也可以不同，完全随机
- 少于5人时，困难任务的次要玩家按实际人数减少（总在场人数-2=次要玩家数）
- 超难任务目标为"所有其他在场玩家"，不显示次要目标栏

### 3.3 难度分级

| 难度 | 积分 | 任务特点 | 惩罚特点 | 任务库 | 惩罚库 |
|------|------|----------|----------|--------|--------|
| 🟢 简单 | +1分 | 聚会中自然可能发生 | 轻微、好笑 | 50条 | 30条 |
| 🟡 中等 | +2分 | 需要一定引导技巧 | 有点尴尬但不难受 | 50条 | 30条 |
| 🔴 困难 | +3分 | 需要演技和套路，容易被识破 | 比较社死或出力 | 50条 | 20条 |
| 💀 超难 | +5分 | 极其离谱，几乎不可能不被发现 | 非常社死 | 25条 | 10条 |

### 3.4 超难任务触发规则

- 💀超难和🔴困难**共用同一个任务栏位**
- 初始手牌3张（🟢🟡🔴），不会开局就拿到超难
- 🔴困难栏位刷新时，**每个栏位独立**20%概率出💀超难，80%出🔴困难
- 🟢简单和🟡中等栏位不会出超难
- 💀超难完成后刷新回困难栏位（仍20%概率出超难）
- 手牌始终3张，超难和困难永远只占同一个栏位

### 3.5 任务库规则

- 任务与惩罚的绑定：**同难度随机绑定**，非固定配对，每次刷新重新随机配对
- 全局共享任务库，不同玩家可以抽到相同任务，允许重复
- 任务库内容见《任务库与惩罚库v2定稿》文档

### 3.6 任务类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 言语诱导 | 引导目标说出特定内容 | "让目标玩家夸你今天好看" |
| 行为引导 | 引导目标做出特定动作 | "让目标玩家主动跟你碰杯" |
| 阻止类 | 阻止目标做出指定行为 | "阻止目标玩家碰杯2次" |
| 情感操控 | 利用社交压力让目标就范 | "让目标玩家当众向你道歉" |

## 四、核心机制

### 4.1 声明完成

```
玩家A线下成功引导目标做出指定行为
  → 点击任务卡上的【完成】按钮
  → 二次确认弹窗："确认完成此任务？"
  → 确认后发送至目标玩家
  → 目标玩家收到确认弹窗
```

**确认弹窗内容：**
- 发起方昵称
- 任务内容
- 惩罚内容
- **不显示任务难度等级**
- 操作：✅ 确认 / ❌ 否认

| 结果 | 效果 |
|------|------|
| ✅ 目标确认 | A获得积分，目标玩家接受惩罚，任务标记【已完成】，任务卡可翻面展示惩罚，动态流公开 |
| ❌ 目标否认 | 触发质疑流程（见4.5），任务标记【被质疑】，执行者接受惩罚 |

**规则：**
- 每条任务只能声明一次，否认后不可重试
- 不发送至次要玩家，不需要次要玩家确认
- 可以同时声明多条任务完成，逐条独立确认

### 4.2 质疑反杀（v2自动化）

```
玩家B感觉某人在害自己（或观察到了线索）
  → 点击【🛡️质疑】按钮
  → 选择质疑目标玩家A
  → 输入猜测的任务内容文本
  → 系统自动匹配A当前3条待完成任务
  → 计算文本相似度（去除模板词后）
  → 相似度≥75% → 质疑成功
  → 相似度<75% → 质疑失败
```

**质疑判定规则：**
- **相似度算法**：去除任务模板固定词（"让目标玩家""让目标玩家及次要玩家""让目标玩家及次要玩家们""让所有玩家"）后，对剩余核心任务内容计算文本相似度
- **75%相似度阈值**：≥75%为质疑成功，<75%为失败
- **多条匹配**：若同时匹配A的多条任务，取相似度最高的一条
- **判定规则可后续优化**：当前75%阈值为基础值，上线后可根据实际体验调整

**质疑结果：**

| 结果 | 效果 |
|------|------|
| 质疑成功 | A的任务标记【被质疑】，A接受惩罚（惩罚涉及多人时，A选择其他人一起接受），质疑者+1刷新次数，A+1刷新次数，任务卡可翻面查看惩罚 |
| 质疑失败 | 无事发生，A任务不变，质疑者无惩罚 |
| 多人同时质疑同一任务 | 先提交的生效，后提交的自动失败无惩罚 |

**规则：**
- 随时可以发起质疑，不需要等对方声明完成
- 只能针对**待完成**状态的任务发起质疑
- 只能看到自己的任务，看不到其他玩家的任务卡牌
- 被质疑成功不扣分

### 4.3 刷新任务（v2重做）

**刷新次数：**
- 初始每人**3次**刷新次数
- 刷新次数**可攒着用**，无上限

**获得刷新次数：**

| 事件 | 刷新次数变化 |
|------|------------|
| 初始进入 | +3次 |
| 3张卡全部resolve（已完成或被质疑） | +3次 |
| 被质疑成功 | 质疑者+1次，任务执行者+1次 |

**刷新流程：**

```
玩家点击【🔄刷新】按钮
  → 消耗1次刷新次数
  → 3张卡同时替换为新任务
  → 每个栏位独立随机：
      🟢简单栏位 → 100%出简单
      🟡中等栏位 → 100%出中等
      🔴困难栏位 → 80%出困难，20%出超难
  → 所有目标同时重新随机分配
  → 已完成或被质疑的卡不进入刷新卡池
```

**刷新次数展示：**
- 刷新按钮旁实时展示剩余刷新次数，如"🔄 3"
- 刷新次数为0时按钮置灰不可点击，hover提示"暂无刷新次数"
- 刷新次数变化时数字做动效（+1飘字等）

### 4.4 目标否认流程（v2新增）

- 执行者点【完成】→ 二次确认 → 发送至目标玩家确认
- 目标玩家**否认** = 触发质疑流程
- 目标否认时：任务标记为【被质疑】，执行者接受惩罚，执行者+1刷新次数，目标玩家（作为质疑者）+1刷新次数

### 4.5 任务状态

| 状态 | 说明 | 卡牌表现 |
|------|------|----------|
| 待完成 | 初始状态 | 正面展示，可操作 |
| 已完成 | 目标确认完成 | 可翻面查看惩罚 |
| 被质疑 | 被质疑成功或目标否认 | 可翻面查看惩罚 |

### 5.2 卡牌颜色

| 难度 | 卡牌底色 | 风格参考 |
|------|----------|----------|
| 🟢 简单 | 白色 | 肉鸽游戏普通卡 |
| 🟡 中等 | 绿色 | 肉鸽游戏稀有卡 |

### 5.3 卡牌交互

- **点击卡牌** → 展开详情（惩罚预览区暂显示"???"）
- **点击【完成】** → 二次确认弹窗："确认完成此任务？"
- **确认后** → 发送至目标玩家确认（不发送至次要玩家，不需要次要玩家确认）
- **目标确认后** → 任务卡可翻面，展示惩罚内容
- **翻牌后** → 立即线下执行惩罚，系统不处理惩罚流程

### 5.4 刷新按钮

- 首页任务卡区域下方设【🔄刷新】按钮
- 按钮旁实时展示剩余刷新次数，如"🔄 3"
- 刷新次数为0时按钮置灰不可点击，hover提示"暂无刷新次数"
- 刷新次数变化时数字做动效（+1飘字等）

## 六、惩罚归属规则（v2新增）

### 6.1 任务完成 → 惩罚对象

| 惩罚文本类型 | 受罚对象 |
|------------|---------|
| 惩罚指定单人执行 | 目标玩家受惩罚 |
| 惩罚涉及多人 | 目标玩家+次要玩家一起受惩罚 |

### 6.2 被质疑成功 → 惩罚对象

| 惩罚文本类型 | 受罚对象 |
|------------|---------|
| 惩罚指定单人执行 | 任务执行者受惩罚 |
| 惩罚涉及多人 | 任务执行者选择其他在场玩家一起受惩罚 |

### 6.3 惩罚执行

- 翻牌后**立即线下执行**，系统不处理惩罚流程
- 惩罚文本不需要统一模板，不与难度目标体系关联

### 7.2 积分展示

**游戏中：**

| 信息 | 可见范围 |
|------|----------|
| 自己的积分 | 仅自己可见 |
| 是否处于后2名 | 仅后2名的玩家可见警告提示 |
| 他人积分 | 不可见 |

**游戏结束：**

| 信息 | 可见范围 |
|------|----------|
| 全员积分排名 | 所有人可见 |
| 积分详情（含历史已完成任务） | 点击积分详情可查看 |

### 7.3 历史记录（v2新增）

- 新增【积分详情】入口
- 点击可查看历史已完成任务列表
- 包含：任务内容、目标玩家、完成时间、获得积分、对应惩罚

### 7.4 动态流

**推送时机：**
- 任务声明完成且目标确认后
- 质疑成功后

**展示内容：**

| 信息 | 是否展示 |
|------|----------|
| 发起方昵称 | ✅ |
| 目标玩家昵称 | ✅ |
| 任务内容 | ✅ |
| 惩罚内容 | ✅ |
| 任务难度 | ❌ 不展示 |
| 任务积分 | ❌ 不展示 |

### 7.5 匿名爆料

**生成规则：**
- 每次有人完成任务时，从全场玩家的当前任务池中随机取2条生成匿名爆料
- 匿名爆料区**至少保持2条**，不足时自动补足
- 开局时即生成2条初始爆料

**内容规则：**
- 模糊化处理，不指明具体玩家和具体动作
- 超难任务的爆料可更"诱人"，如"💡 有人拿到了一个值5分的危险任务……"

**消失规则：**
- 对应任务被刷新/完成后，该条匿名爆料消失
- 消失后自动补足至2条

**示例：**
- "💡 有人想让三位玩家做同一件事…"
